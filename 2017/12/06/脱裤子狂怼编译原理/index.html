正则表达式

正则表达式实际上是用于描述一套字符串集合所定义的一套语法，我们利用特定的字符串加上一些符号去描述我们想描述的一些具有某些性质的字符串。
为什么需要正则表达式呢，大概是因为在描述一些数量庞大的字符串集合的时候，应用正则表达式的概念，会让我们的描述更加清晰。
基本可以分为3种：

ab，连接
a | b，或
a*，0到多个a，例aaaaa

为了让机器理解正则表达式，我们需要通过程序来实现一种与正则表达式等价的结构来实现：非确定性有穷自动动和确定性有穷自动机。

NFA(非确定有穷自动机)

NFA抽象地表示了用来识别某个语言中的串的算法(即识别串)。
一个不确定的有穷自动机(NFA)会由以下几个部分组成：

  1. 一个有穷的状态集合S。
  2. 一个输入符号集合∑，即输入字母表(input alphabet)。
  3. 一个转换函数(transition function)，它为每个状态和∑ U {ϵ}中的每个符号都给出了相应的后继状态(next state)的集合。
  4. S中的一个状态S0被指定为开始状态，或者说初始状态。
  5. S的一个子集F被指定为接受状态(或者说终止状态的)集合。

其实说明了就是对路径进行穷举遍历。

DFA(确定有穷自动机)

相应的DFA则是一个简单具体的识别串的算法(具体的识别算法)。
确定有穷自动机(DFA)和不确定有穷自动机的区别：

  1. 没有输入ϵ之上的转换动作。
  2. 对每个状态s和每个输入符号a，有且只有一条标号为a的边离开s。

从NFA转换到等价的DFA需要满足如下条件：

  • 状态集合：subsets of S，其中S是状态集合。
  • 弧集合：move(t, a)，是指t中某一状态经过一条α弧到达的状态的集合。
  • 初始状态：ε-closure(s)，是指s中的任何状态s经过任意条ε弧能到达的状态的集合。
  • 终止状态：{x | x ∧ F ≠ ϕ}，其中F是NFA的终止状态集合。
  • 转换集合：X.a->Y， Y = ε-closure(move(X, a))。

3.语法分析

根据前面我们已经了解到，语法分析器通过调用getNextToken函数获取一个个的token之后对其进行处理，并验证文法规则。因为其对源程序的检查和翻译动作可以和语法分析过程交错完成，所以，语法分析器和前端的其他部分可以只用一个模块来实现。
如果我们把词法分析看作是拆分单词归类的一个过程，那么语法分析就是一个把单词组合成句子的过程。(脱衣服洗完澡又穿衣服一个过程)

可以使用现成的工具直接生成：yacc等。

上下文无关文法(context-free grammar, CFG)

在前面词法分析时我们是用正则表达式来描述词法规则的，而在语法分析中，我们需要使用一种叫作’上下文无关文法’(简称文法)的方式来描述语法规则。
文法优点：能够更精确、形式化的描述。
我们可以把一种程序开发语言看作是无数个符合语法规则的句子集合，然后根据给定的文法我们可以判断指定的token串是否符合某个语法规则，如果符合，那么我们可以把此文法和对应输入的token串组合起来生成一个句子。

一个文法由终结符号、非终结符号、一个开始符号和一组产生式组成，即一个四元组（N，T，S，P）：

  1. N，终结符号是组成串的基本符号。在前面例子中，终结符号是关键字if和else以及符号”(“和”)”。
  2. T，非终结符号是表示串的集合的语法变量。在前面例子中，stmt和expr是非终结符号。表示的串集合用于定义由文法生成的语言，非终结符号给出了语言的层次结构，而这种层次结构是语法分析和翻译的关键。
  3. S，在一个文法中，某个非终结符号被指定为开始符号。这个符号表示的串集合就是这个文法生成的语言。
  4. P，一个文法的产生式描述了将终结符号和非终结符号组成串的方法。

每个产生式由下列元素组成：

  • 一个被称为产生式头或左部的非终结符号。这个产生式定义了这个头所代表的串集合的一部分。
  • 符号->。有时也使用::=来替代箭头。
  • 一个由零个或多个终结符号与非终结符号组成的产生式体或右部。产生式体中的成分描述了产生式头上的非终结符号所对应的串的某种构造方法。

文法规则

终结符号：

  1. 在字母表里排在前面的小写字母，比如a、b、c。
  2. 运算符号，比如+、-、*、/等。
  3. 标点符号，比如括号、逗号等。
  4. 数字0、1、…、9。
  5. 黑体字符串，比如id或if。

非终结符号：

  1. 在字母表中排在前面的大写字母，比如A、B、C。
  2. 字母S，它出现时通常表示开始符号。
  3. 小写、斜体的名字，比如expr或stmt。
  4. 当讨论程序设计语言的构造时，大写字母可以用于表示代表程序构造的非终结符号。比如，表达式、项和因子的非终结符号通常分别用E、T和F表示。

在字母表中排在后面的大写字母(X、Y、Z)表示文法符号。即表示非终结符号或终结符号。
在字母表中排在后面的小写字母(u、v、…、z)表示(可能为空的)终结符号串。
小写的希腊字母，αβγ表示(可能为空的)文法符号串。所以，一个普通的产生式可以写成A->α，其中A是产生式的头，α是产生式的体。
具有相同的头的一组产生式A->α1，A->α2，…，A->ak(A产生式)可以写作A->α1|a2|…|ak|。我们把a1，a2，…，ak称作A的不同可选体。
除非特别说明，第一个产生式的头就是开始符号。
简单算术表达式的文法例子：终结符号是：id + - * / ()， 非终结符号expression、term和factor，而expression是开始符号

推导

将产生式看作重写规则，就可以从推导的角度精确地描述构造语法分析树的方法。从开始符号出发，每个重写步骤把一个非终结符号替换为它的某个产生式的体。这个推导思想对应于自顶向下构造语法分析树的过程，但是推导概念所给出的精确性在讨论自底向上的语法分析过程时尤其有用。自底向上语法分析和一种被称为”最右”推导的推导类型相关。
我们看例子：

E->-E，即E=>-E，读作”E推导出-E”

产生式E->(E)可以将任何文法符号串中出现的E的任何实例替换为(E)，如：

E E =>(E) E或E E=>E (E)。

我们可以按照任意顺序对单个E不断应用各个产生式，得到一个替换的序列。如：

E=>-E=>-(E)=>-(id)

我们把这个替换序列称为从E到-(id)的推导。这个推导证明了串-(id)是表达式的一个实例。
符号=>，表示通过一步推导出。
符号=>，表示经过零步或多步推导出。
符号=>+，表示经过一步或多步推导出。
如果S=>a，其中S是文法G的开始符号，我们说a是G的一个句型(sentential form)。请注意，一个句型可能既包含终结符又包含非终结符号，也可能是空串。文法G的一个句子(sen-tence)是不包含非终结符号的句型。一个文法生成的语言是它的所有句子的集合。所以，一个终结符号w在G生成的语言L(G)中，w只是G的一个句子(或者说S=>*w)。可以由文法生成的语言被称为上下文无关语言(context-free language)。如果两个文法生成相同语言，这两个文法就被称为是等价的。
在每个推导步骤中按照如下方式选择被替换的非终结符号的两种推导过程：

  1. 在最左推导(leftmost derivation)中，总是选择每个句型的最左非终结符号。
  2. 在最右推导(rightost derivation)中，总是选择最右边的非终结符号。

如果S=>a*lm，我们说a是当前文法的最左句型(left-sentential form)。
最右推导有时也称为规范推导(canonical derivation)。

扯了那么多难免有点抽象，下面我们来看一个例子：

S –> AB
A –> aA | ε
B –> b | bB

其中S，A，B就是非终结符，代表可以继续扩展或产生的符号；a b ε是终结符，表示其无法再产生新的符号了，其中ε表示一个空句子；上面的每一行就是一个产生式规则，代表了一种非终结符的转移方式；而S就是开始符号。
只有终结符的符号串称为 句子（sentence）。经过观察可知，这个语法所能推导出的所有句子的集合为：

A : { ε, a, aa, aaa, … }
B : { b, bb, bbb, … }
S : { b, bb, bbb, …, ab, abb, …, aab, aabb, … }

语法制导翻译

语法制导翻译（Syntax-directed translation，SDT），是在解析（parse）输入的Token时，给每一个产生式规则附加一个语义动作（一个代码片段），即把语法“翻译”成一串“动作”，故名“语法制导翻译”。
例如，我们可以在语法动作中加入生成抽象语法树的行为。

分析树和抽象语法树

语法分析不但要判断给定的句子是否符合语法结构，而且还要分析出该句子符合哪些结构，也就是说，要分析出这个句子是怎么从起始符号开始产生出来的，并根据产生过程生成语法树。例如，对于 “我吃饭” 这句话，单纯的知道这句话符合某一个CFG是没有意义的，只有我们知道了 主语、谓语和宾语 分别对应的是那个词，我们才能真正理解这句话的含义。
抽象语法树(abstract syntax tree或者缩写为AST)，是源代码的抽象语法结构的树状表现形式。其实就是把源码弄成树型图。

分析算法

下面介绍几种常用的算法：

  • 自顶向下分析算法
  • 自底向上分析算法
  • 递归下降分析算法
  • LL(1)分析算法
  • LR(1)分析算法

自顶向下分析算法

从根结点开始，按照先根次序创建这棵语法分析树的各个结点。自顶向下语法分析也可以被看作寻找输入串的最左推导的过程。
当发现某一个产生式产生的终结符和句子的当前结构根本不匹配时，我们就不会再向下继续分析了，因为不管怎么分析也无法得到正确的结果。此时我们就需要回到最近的一次有效的规则，然后继续使用新的规则向后执行，这种行为称为“回溯”。

递归下降分析算法

递归下降分析算法本质也是一种自顶向下分析算法，其基本思想如下：

  • 对每一个非终结符构造一个分析函数
  • 用前看符号指导产生式规则的选择

递归下降分析算法使用“分治法”来提高分析的效率，对于每一个产生式规则，都应该定义一个自己函数。因为在上下文无关文法中，终结符不可能出现在产生式的左边（可以在产生式左边出现终结符的文法叫做上下文有关文法），上下文无关文法中所有的产生式左边只有一个非终结符。所以我们在调用产生式规则的函数后，就分为两种情况：

  1. 遇到终结符，因为终结符本质上是token，所以直接把这个终结符和句子中对应位置的token进行比较，判断是否符合即可；符合就继续，不符合就返回
  2. 遇到非终结符，此时只需要调用这个非终结符对应的函数即可。在这里函数可能会递归的调用，这也是算法名称的来源。

简单来说，就是遇到非终结符就调用函数，遇到终结符就比较；

LL(1)分析算法

LL(1)算法也是一个自顶向下的分析算法，它的定义为：从左（L）向右读入一个程序，最左（L）推导，采用一个（1）前看符号。LL(1)算法和自顶向下分析算法本质上是一致的，它们的区别就在于LL(1)算法使用了一种称为分析表的工具来避免了回溯操作，提高了效率。在现实中，我们可以根据某种LL(1)分析器来生成分析表，之后根据分析表来进行语法分析操作，我们可以认为这种方法是一种半自动的语法分析方法。

小结：上面我们讲了LL(1)算法的主要内容，需要理解LL(1)算法其实就是对于每个非终结符，能够通过查阅分析表找到正确的产生式规则，这种类似于“偷看”的策略能够避免“回溯”操作，可以有效的提高语法分析的效率。
需要注意的是，LL(1)文法不能拥有左递归，否则会导致递归到死，所以如果使用LL(1)文法则需要消除左递归。

自底向上分析算法的主要方式是根据句子的Token，使用产生式来自右向左进行分析。

目前最流行的自底向上语法分析器都是基于LR(k)语法分析的概念，”L”表示对输入进行从左到右的扫描，”R”表示反向构造一个最右推导序列，而k表示在做出语法分析决定时向前看k个输入符号。

LR语法分析表由一个语法分析动作函数ACTION和一个转换函数GOTO组成：

  1. ACTION函数有两个参数：一个状态i，另一个是终结符号a(或者是输入结束标记$)。ACTION[i, a]取值可以有下列四种形式：
    • 移入(shift)j，其中j是一个状态。
    • 归约(reduce)。
    • 接受(ACCEPT)。
    • 报错。
  2. 我们把定义在项集上的GOTO函数扩展为定义在状态集上的函数：如果GOTO[Ii, A] = Ij，那么GOTO也把状态i和一个非终结符号A映射到状态j。

LR(1)

LR(0)要求文法状态中不能包含多条可折叠项、且不能同时有可折叠项和不可折叠项，这是为了避免reduce/reduce冲突和shift/reduce冲突，此限制条件相当强，导致LR(0)的适用范围非常小。这时候就需要用到LR(1)了，跟LR(0)差不多，但是多了lookahead向前看1字节的操作。

LR(1)语法要求为：

  1. 起始符号S不能位于任何产生式的右边。
  2. 从此文法的起始状态开始生成的所有状态中。
    1. 任何一个状态不能同时含有[A -> u·aw，c]和[B -> v·，a]。
    2. 任何一个状态不能同时含有[A -> u·，a]和[B -> v·，a]。

LR(1)特点，添加lookahead，格式表示成下面这样：

[S -> ·A, $]

这个$就是lookahead的意思。

其实LR(1)跟LR(0)的操作是基本类似的，只是归约加了条件。
I中含有项[A-> X1、X2、…、Xn·, X]，有以下两种情况：

  1. A != S或X != $： M[I, X] = reduce A -> X1、X2、…、Xn。
  2. A == S且X == $： M[I, X] = accept。

简单LR(1)分析，对LR(0)的优化版，LR(1)简化版。

4.语义分析

语义分析也称为类型检查、上下文相关分析，它负责检查程序（抽象语法树）的上下文相关的属性。
语义分析的操作十分依赖于该语言的语法格式，不同语言的语义分析有可能完全不一样。

语法制导定义

语法制导定义(Syntax-Directed Definition，SDD)是一个上下文无关文法和属性及规则的结合。属性和文法符号相关联，而规则和产生式相关联。

综合属性

综合属性(synthesized attribute)：在分析树结点N上的非终结符号A的综合属性是由N上的产生式所关联的语义规则来定义的。需要注意的是，这个产生式的头一定是A。结点N上的综合属性只能通过N的子结点或N本身的属性值来定义。即只能从子结点或本身获取其值。
举个例子：

S -> ABC

如果S从其子结点（A，B，C）获取值，则称为综合属性，因为ABC的值合成了S。

继承属性

继承属性(inherited attribute)：在分析树结点N上的非终结符号B的继承属性是由N的父结点上的产生式所关联的语义规则来定义的。需要注意的是，这个产生式的体中必然包含符号B，结点N上的继承属性只能通过N的父结点、N本身和N的兄弟结点上的属性值来定义。(与综合属性相反，可以从其父结点和兄弟结点中获取值)。
同样上面的例子：

S -> ABC

A可以从S、B和C获得值，同理B可以从S、A和C中获取属性值。C同上。

SDT

语法制导的翻译方案(syntax-directed translation scheme，SDT)是在其产生体中嵌入了程序片段的一个文法。这些程序片段称为语义动作，它们可以出现在产生体中的任何地方。
语义动作规则：

  • 动作两边添加花括号。
  • 如果要作为文法符号出现，则要给它们加上引号。

所有SDT(需要前序遍历执行)步骤：

  1. 建立一棵语法分析树。
  2. 按照从左到右的深度优先顺序来执行语义动作。

一般情况下，SDT是在语法分析过程中实现的，不会真的去构造一棵语法分析树。
使用SDT来实现两类重要的SDD：

  1. 基本文法可以用LR技术分析，且SDD是S属性的。
  2. 基本文法可以用LL技术分析，且SDD是L属性的。

如上所述，S属性SDT中的属性在自下而上解析中进行评估，因为父节点的值取决于子节点的值。
我们可以得出结论，如果一个定义是S归因的，那么它也是L归因的，因为L属性定义包含S属性定义。

需要注意并检查的逻辑

  1. 重复声明，定义。
  2. 未定义，未声明。
  3. 下标非法，访问冲突。
  4. 类型冲突不符合。
  5. 其它。

5.中间代码生成

中间代码表示形式：

  1. 无环有向图(Directed Acyclic Graph，简称DAG)，指出了表达式中的公共子表达式
  2. 三地址
  3. 四元式
  4. 三元式，间接三元式
  5. 静态单赋值形式

DAG

和表达式的语法树类似，一个DAG的叶子结点对应于原子运算分量，而内部结点对应于运算符。与语法树不同的是，如果DAG中的一个结点N表示一个公共子表达式，则N可能有多个父结点。(就是把运算分量放在叶子，内部结点放运算符)
语法树或DAG图中的结点通常存放在一个记录数组中，数组的每一行表示一个结点，由一个标号、左右结点组合而成，访问的时候，只需要给出此数组中的整数下标就可以引用该结点，我们称这个整数为相应结点或该结点所表示的表达式的值编码(value number)。

三地址

三地址代码比较简单，一条指令的右侧最多有一个运算符。其基于两个基本概念：地址和指令，地址可以具有如下形式之一：

  • 名字， 允许源程序的名字作为三地址代码中的地址。
  • 常量。
  • 编译器生成的临时变量。

常见的三地址指令形式：

  1. x = y op z， 其中op是运算符，x、y、z是地址。

四元式

一个四元式(quadruple)由四个字段组成：op、arg1、arg2、result。字段op包含一个运算符的内部编码。
举个例子(三地址)：

x = y + z

如果要用四元式去表示，op存放+，arg1=y，arg2=z，result=x。
这时候，有的小伙伴会问了，那单目运算符指令x+=1，x = y的指令怎么办，对于这类处理是不使用arg2。
那call x，return x之类怎么处理，直接不使用arg2，也不使用result。
那条件或非条件转移指令之类怎么处理，直接将目标标号放入result。
(丢，这还是四元式吗?)

三元式

三元式(triple)只有三个字段组成：op、arg1和arg2，四元式区别是，没有result字段。
三元式直接用运算式x op y的位置来表示它的结果(丢，这不就是它本身吗?)。

间接三元式

由于使用三元式时，对于运算结果的引用是通过位置完成的。因此如果改变一条指令的位置，则引用该指令的结果的所有指令都要做相应的修改。这时候就需要用到间接三元式(indirect triple)了，其包含了一个指向三元式的指针的列表，而不是列出三元式序列本身(其实就是加一层指针，维护到数组里)。

6.优化

优化是在不改变程序运行效果的前提下对被编译的程序进行优化改变，目的是使其能生成更简洁快速的目标代码。
优化方法：

  1. 干掉”冗余代码”(即用不上未曾使用的代码可以把它给干掉)。
  2. 内嵌。可以直接缩减
  3. 公共子表达式可以直接缩减。比如x = 1+2+3+4，可以直接表示为x = 10。
  4. 常量。
  5. 代码外提。
  6. 强度削弱。
  7. 删除归纳变量。
